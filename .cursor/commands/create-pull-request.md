# GitHub プルリクエスト作成

## 概要

現在のブランチからプルリクエストを作成します。

## 使用方法

このコマンドを使用する際は、以下の情報を提供してください：
- **ベースブランチ**（オプション）: デフォルトは`develop`です

## 手順

1. **現在のブランチ情報を確認**
  - `git branch` で現在のブランチを確認
  - `git log` でコミット履歴を確認

2. **変更内容の確認**
  - `git diff develop...HEAD` で変更内容を確認
  - 変更されたファイルをリスト化

3. **プルリクエスト本文の作成**
  - 変更内容を分析し、適切なPR本文を作成
  - 関連するissue番号を含める

4. **GitHub CLIでプルリクエストを作成**
  - タイトル: 変更内容を簡潔に表現
  - 本文: 上記で作成した内容を日本語で記載

## プルリクエスト本文の構成

以下の構成でPR本文を作成してください：

```markdown
## 変更内容
[このPRで何を変更したか]

## 変更理由
[なぜこの変更が必要だったか]

## 実装詳細
[実装の詳細や技術的な説明]

## 関連Issue
[関連するissue番号]
Closes #<issue番号>

## チェックリスト
- [ ] コードレビューを実施した
- [ ] テストを追加/更新した（必要に応じて）
- [ ] ドキュメントを更新した（必要に応じて）
- [ ] 型チェックが通る（`pnpm check:type`）
- [ ] リントチェックが通る（`pnpm lint`）
- [ ] ビルドが成功する（`pnpm build`）
```

## プルリクエストタイトルの形式

以下の形式でタイトルを作成してください：

```
<type>(<scope>): <subject>
```

例：
- `feat(todo): add todo list component`
- `fix(api): fix batch processing failed URL status`
- `refactor(api): extract common batch processing logic`

## 変更内容の分析

PR本文を作成する際は、以下の観点で変更内容を分析してください：

1. **主な変更**: 何が変更されたか
2. **変更の理由**: なぜ変更されたか
3. **実装詳細**: どのように実装されたか
4. **影響範囲**: どの部分に影響があるか
5. **関連ファイル**: どのファイルが変更されたか

## 注意事項

- **日本語で記述**: PR本文は日本語で書いてください
- **署名不要**: PR本文にClaude Codeによるものであることがわかる署名は不要です
- **関連issue**: 関連するissue番号を必ず含めてください（`Closes #<issue番号>`の形式）
- **チェックリスト**: 適切なチェックリストを含めてください
- **想定外の作業**: 想定外の作業が発生したり、エラーが見つかった場合はユーザに相談してください
- **リモートプッシュ**: ブランチがリモートにプッシュされていることを確認してください

## チェックリスト

作成後、以下を確認してください：

- [ ] 現在のブランチ情報が正しく取得できている
- [ ] 変更内容が適切に分析されている
- [ ] PRタイトルが適切に設定されている
- [ ] PR本文が上記の構成に従って作成されている
- [ ] 関連するissue番号が含まれている
- [ ] チェックリストが適切に設定されている
- [ ] 日本語で記述されている
