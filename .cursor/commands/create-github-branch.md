# GitHub ブランチ作成

## 概要

指定されたissue番号に対して、適切なブランチ名でブランチを作成し、チェックアウトします。

## 使用方法

このコマンドを使用する際は、以下の情報を提供してください：
- **Issue番号**: 対応するGitHub issueの番号

例：
```
Issue番号: 56
```

## 手順

1. **Issue情報の取得**
  - GitHub CLI (`gh`) を使用してissue情報を取得
  - issueのタイトルと内容を確認

2. **ブランチ名の決定**
  - ブランチ名のルールに従って適切なブランチ名を作成
  - 機能追加の場合: `feature/{issue番号}-{課題の内容を表す英語名}`
  - バグ修正の場合: `fix/{issue番号}-{課題の内容を表す英語名}`

3. **ブランチの作成とチェックアウト**
  - `git checkout -b <ブランチ名>` でブランチを作成してチェックアウト

4. **リモートへのプッシュ（オプション）**
  - 必要に応じて `git push -u origin <ブランチ名>` でリモートにプッシュ

## ブランチ名のルール

### 機能追加の場合
```
feature/{issue番号}-{課題の内容を表す英語名}
```

例：
- `feature/56-add-user-profile`
- `feature/42-implement-todo-list`

### バグ修正の場合
```
fix/{issue番号}-{課題の内容を表す英語名}
```

例：
- `fix/56-batch-processing-failed-url-status`
- `fix/42-todo-list-rendering-bug`

### 命名のガイドライン
- **kebab-case**: すべて小文字で、単語はハイフンで区切る
- **簡潔に**: ブランチ名は簡潔で分かりやすく
- **英語**: ブランチ名は英語で記述
- **issue番号を含める**: 必ずissue番号を含める

## 注意事項

- **現在のブランチ**: ブランチ作成前に現在のブランチを確認し、適切なベースブランチ（通常は`main`または`develop`）にいることを確認してください
- **未コミットの変更**: 未コミットの変更がある場合は、ユーザーに確認してください
- **想定外の作業**: 想定外の作業が発生したり、エラーが見つかった場合はユーザに相談してください

## チェックリスト

作成後、以下を確認してください：

- [ ] issue情報が正しく取得できている
- [ ] ブランチ名がルールに従って作成されている
- [ ] ブランチが正しく作成され、チェックアウトされている
- [ ] 必要に応じてリモートにプッシュされている
