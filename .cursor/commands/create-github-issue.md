# GitHub Issue作成

## 概要

ユーザーから要求された内容に対して、適切なGitHub issueを作成します。

## 使用方法

このコマンドを使用する際は、以下の情報を提供してください：
- **要求内容**: 実装したい機能や修正したいバグの説明

例：
```
要求内容: 特定のURLが要約処理失敗になった場合に、結果に含まれないためステータスが処理中のままとなるバグの修正
```

## 手順

1. **現在のリポジトリ情報を確認**
  - GitHub CLI (`gh`) を使用してリポジトリ情報を取得

2. **Issue内容の作成**
  - 問題の概要と現在の動作を説明
  - 期待される動作を明記
  - 影響範囲を記載
  - 可能であれば修正案を提示
  - 関連ファイルをリスト化（既存のコードベースを確認）

3. **GitHub CLIでissueを作成**
  - タイトル: 要求内容を簡潔に表現
  - 本文: 上記で作成した内容を日本語で記載

## Issue本文の構成

以下の構成でissue本文を作成してください：

```markdown
## 問題の概要
[問題の説明]

## 現在の動作
[現在どのような動作をしているか]

## 期待される動作
[どのような動作が期待されるか]

## 影響範囲
[この問題が影響する範囲]

## 修正案
[可能であれば修正案を提示]

## 関連ファイル
[関連するファイルのリスト]
```

## 注意事項

- **日本語で記述**: issueの内容は日本語で書いてください
- **署名不要**: issueの内容にClaude Codeによるものであることがわかる署名は不要です
- **想定外の作業**: 想定外の作業が発生したり、エラーが見つかった場合はユーザに相談してください
- **リポジトリ確認**: 必ず現在のリポジトリ情報を確認してからissueを作成してください

## チェックリスト

作成後、以下を確認してください：

- [ ] リポジトリ情報が正しく取得できている
- [ ] issueのタイトルが適切に設定されている
- [ ] issue本文が上記の構成に従って作成されている
- [ ] 関連ファイルが適切にリスト化されている
- [ ] 日本語で記述されている
